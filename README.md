# Tri-M Global Logistics & Trading Inc. — Workforce360

Cloud-based workforce management system: biometric attendance, route scheduling,
and compliance analytics. React + TypeScript frontend, Supabase (Postgres + Auth) backend.

This rebuilds the Figma "Cloud-Based Workforce Management System" design, rebranded
for **Tri-M Global Logistics & Trading Inc.**

## 2. Run it locally

Requirements: **Node.js 18+** and **npm 9+**.

```bash
cd tri-m-workforce
npm install
cp .env.example .env      # then fill in your Supabase values (step 3)
npm run dev               # http://localhost:5173
```

Build for production:
```bash
npm run build
npm run preview
```

## 3. Set up Supabase (backend + database)

1. Create a free project at [supabase.com](https://supabase.com).
2. In **Project Settings → API**, copy the **Project URL** and **anon public key**
   into your `.env`:
   ```
   VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGciOi...
   ```
3. Open **SQL Editor → New query**, paste the entire contents of
   `supabase/schema.sql`, and run it. This creates:
   - `agencies`, `principals`, `merchandisers`, `stores`
   - `attendance_logs`, `leave_requests`, `routes`
   - Indexes for fast dashboard queries
   - **Row Level Security (RLS)** policies so each agency only sees its own data
   - One seed row for Tri-M Global Logistics & Trading Inc. and three sample principals
4. (Optional) Enable **Email** auth under **Authentication → Providers** — it's on
   by default. Turn off "Confirm email" while testing locally so signup is instant.
5. (Optional) Insert demo attendance data:
   ```bash
   SUPABASE_SERVICE_ROLE_KEY=your-service-role-key \
   VITE_SUPABASE_URL=https://xxxxxxxx.supabase.co \
   node scripts/seed.mjs
   ```
   Get the service role key from **Project Settings → API** — keep it server-side only,
   never commit it or expose it in frontend code.

### Linking real merchandiser logins
Each row in `merchandisers` has a `user_id` column. After a field merchandiser
signs up via Supabase Auth, update their row:
```sql
update merchandisers set user_id = '<their-auth-uid>' where id = '<merchandiser-id>';
```
This is what the RLS policies use to scope what each user can read.

## 4. Deploying

### Frontend — Vercel (recommended)
1. Push this repo to GitHub.
2. In [vercel.com](https://vercel.com), "Add New Project" → import the repo.
3. Framework preset: **Vite**. Build command `npm run build`, output dir `dist`.
4. Add environment variables in Vercel's dashboard (Settings → Environment Variables):
   `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`.
5. Deploy. Every push to `main` auto-redeploys.

Netlify works the same way (build command `npm run build`, publish dir `dist`).

### Backend
Supabase is already hosted — there is no separate backend server to deploy. Postgres,
Auth, and (if you add them) Storage/Edge Functions all live in your Supabase project.

### Custom domain
Point your domain's DNS to Vercel/Netlify, then add it under Project → Domains.
Update the CORS/allowed URLs in Supabase → **Authentication → URL Configuration** to
include your production domain so login redirects work.

## 5. Integrating with other systems

Because Supabase exposes a standard Postgres database and a REST/GraphQL API on
top of it, other systems (payroll, ERP, principal portals) can integrate in a few ways:

**A. REST API (auto-generated by Supabase / PostgREST)**
```bash
curl "https://xxxxxxxx.supabase.co/rest/v1/attendance_logs?select=*&status=eq.late" \
  -H "apikey: <anon-or-service-key>" \
  -H "Authorization: Bearer <anon-or-service-key>"
```
Any system that can make an HTTP request (payroll software, a BI tool, Zapier/Make)
can read or write rows this way, subject to RLS.

**B. Webhooks (Database Webhooks)**
In Supabase → **Database → Webhooks**, trigger an HTTP call whenever a row is
inserted/updated — e.g. notify a payroll system the moment a timesheet is finalized.

**C. Scheduled exports (Edge Functions + Cron)**
Use Supabase Edge Functions with `pg_cron` to generate and email a nightly CSV of
attendance for principal billing.

**D. Direct Postgres connection**
For BI tools (Metabase, Looker, PowerBI) or a legacy ERP, connect directly using the
Postgres connection string in **Project Settings → Database**. Use a read-only role
for reporting tools.

**E. Single Sign-On**
If Tri-M Global Logistics & Trading Inc. already uses Google Workspace or Microsoft 365,
enable that provider under **Authentication → Providers** in Supabase — no extra
frontend code is required beyond swapping the sign-in call.

## 6. Security checklist before going live

- [ ] Row Level Security is **enabled** on every table (already done in `schema.sql`)
- [ ] Review and tighten the RLS policies for your real role model (HR admin vs.
      field merchandiser vs. principal viewer) — the shipped policies are a starting point
- [ ] Never expose the **service role key** in frontend code or git history
- [ ] Turn on **email confirmation** in Supabase Auth before production launch
- [ ] Set up Supabase **daily backups** (Project Settings → Database → Backups)
- [ ] Add rate limiting / captcha on the login form if it's public-facing
- [ ] Rotate the anon key if it's ever accidentally leaked (Project Settings → API)

## 8. Common errors and fixes

| Symptom | Fix |
|---|---|
| Blank page, console shows "Missing Supabase environment variables" | You forgot to create `.env` from `.env.example`, or forgot to restart `npm run dev` after editing it |
| Login succeeds but dashboard shows 0s everywhere | No rows yet in `attendance_logs`/`leave_requests` — run `scripts/seed.mjs` or insert manually in Supabase Table Editor |
| "new row violates row-level security policy" | The logged-in user's `merchandisers.user_id` isn't linked yet, or you're inserting into a table the anon/authenticated role doesn't have a policy for |
| `tsc` errors about `import.meta.env` | Make sure `src/vite-env.d.ts` exists (it's included) and `tsconfig.json` targets ES2020+ |
| Tailwind classes not applying | Confirm `tailwind.config.js` `content` globs match your file paths, and `index.css` has the three `@tailwind` directives |
